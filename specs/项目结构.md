# 项目结构

## 1. 目录树 (Directory Tree)
```
/
├── specs/                          # 项目核心定义 (产品/技术/结构/规范)
│   ├── 产品概述.md
│   ├── 技术栈.md
│   └── 项目结构.md
├── docs/                           # 过程性文档 / 设计记录 / 决策日志（可选）
├── src/
│   ├── app/                         # Next.js App Router（路由与页面）
│   ├── modules/                     # 业务模块（按核心板块高内聚组织）
│   │   ├── about/                   # “关于我”领域：简历/介绍数据模型与取数
│   │   ├── projects/                # “作品集”领域：项目列表/详情的数据模型与取数
│   │   ├── blog/                    # “博客”领域：文章列表/详情、分类/标签等
│   │   ├── search/                  # “站内搜索”领域：索引产物读取、搜索逻辑
│   │   └── publishing/              # “本地发布/编辑”领域：本地编辑器与发布链路相关能力
│   ├── shared/                      # 跨模块共享（不含业务语义）
│   │   ├── components/              # 通用 UI 组件（Button、Dialog…）
│   │   ├── styles/                  # 全局样式、排版 token
│   │   ├── lib/                     # 工具函数、适配器、第三方封装
│   │   ├── types/                   # 全局类型
│   │   └── config/                  # 站点级配置（站点名、链接、导航等）
│   └── content/                     # 内容层封装：读取/解析 MD(X)、frontmatter、类型化输出
├── content/                         # 内容源（Git 管理，构建期读取）
│   ├── about/                       # 关于我：结构化数据或 MD
│   ├── projects/                    # 作品集条目（MD/MDX + 元数据 + 资源引用）
│   └── blog/                        # 博客文章（MD/MDX + 元数据）
├── public/                          # 静态资源（头像、项目封面、og 默认图等）
├── scripts/                         # 构建期/本地工具脚本（生成索引、校验内容等）
├── tests/                           # 测试（unit / e2e，后续逐步补齐）
├── .gitignore
├── .env.example
├── README.md
├── package.json
├── pnpm-lock.yaml
├── next.config.*                    # Next.js 配置（如需）
├── tailwind.config.*                # Tailwind 配置（如需）
├── tsconfig.json
└── eslint.config.*                  # ESLint 配置（如需）
```

## 2. 关键文件说明 (Key Files)
- **package.json**: 依赖与脚本入口（本地开发、构建、发布）。
- **.env.example**: 环境变量示例（不放任何敏感信息；若仅静态站可保持最小）。
- **README.md**: 项目入口文档，至少包含“如何本地启动”“如何发布到 Vercel”。
- **content/**: 内容源目录（文章/作品/关于我），是“本地可视化编辑 + 一键发布”的核心资产。
- **scripts/**: 构建期产物生成（如搜索索引）、内容校验（frontmatter 完整性）等工具脚本。

## 3. 模块说明 (Module Description)
- **src/modules/about**: 关于我/简历信息的结构化模型与读取逻辑。
- **src/modules/projects**: 作品集条目模型、列表/详情所需的查询与排序逻辑。
- **src/modules/blog**: 博客文章模型、列表/详情所需的查询与排序逻辑。
- **src/modules/search**: 搜索索引的读取与检索逻辑（前端搜索库的薄封装）。
- **src/modules/publishing**: 本地编辑器与“一键发布”相关的能力（仅本地启用的路由、校验、写入流程等）。
- **src/shared**: 纯通用能力（UI/工具/类型/配置），禁止放“博客/作品集”等具体业务规则。

## 4. 文件放置规则 (Placement Rules)
- **页面/路由**：只放在 `src/app/`。页面只负责组合与展示，不直接写业务规则与数据整形。
- **业务逻辑**：一律放在 `src/modules/*` 内（例如排序、过滤、内容聚合、搜索参数解析等）。
- **内容解析/读取**：统一走 `src/content/`（对 `content/` 的读取、MDX 编译、frontmatter 校验）。
- **通用组件**：放在 `src/shared/components/`；可复用样式/排版放在 `src/shared/styles/`。
- **站内搜索索引**：
  - 索引**生成脚本**放 `scripts/`（构建期执行）
  - 索引**产物**建议输出到 `public/search-index/` 或 `.next` 构建产物中（具体实现阶段再定）
- **本地编辑器**：路由建议放 `src/app/(local)/editor`，并通过环境判断确保仅 `development` 可用。
