# About（关于我）— 技术设计文档

## 1. 设计概要

**功能描述**：提供一个极简文字优先的中文 About 页面，用结构化内容展示作者的核心介绍信息，并提供“去博客”和“联系我”的明确下一步行动。→ AC-001~AC-010

**影响范围**：
- 路由与页面：`src/app/about/page.tsx` → AC-001~AC-010
- About 领域模块：`src/modules/about/*`（新增）→ AC-002, AC-003, AC-005~AC-010
- 内容源：`content/about/about.json`（新增）→ AC-002, AC-003, AC-005~AC-010

**技术难点**：低（静态页面 + 内容文件读取）；需要把“缺失字段的兜底展示”设计清楚。→ AC-006, AC-007, AC-010

**外部依赖**：无（不依赖外部服务/后端/数据库）。→ AC-009

---

## 2. 架构概览

About 属于“内容文件在仓库 + 构建期读取 + 静态输出”的典型路径（符合 `specs/技术栈.md` 约束）。

- `content/about/about.json` 作为唯一内容源（作者极少更新，结构化更稳定）→ AC-002, AC-003, AC-005~AC-010
- `src/modules/about/getAbout.ts` 负责读取与最小校验（缺字段时提供默认值/兜底策略）→ AC-006, AC-007, AC-010
- `src/app/about/page.tsx` 负责按 BR-002 固定顺序渲染，并提供页面级 Metadata → AC-001~AC-004, AC-008~AC-010

## 3. 数据库设计

本功能不引入数据库：数据全部来自仓库内内容文件。→ AC-009

本功能不新增 API：About 为静态页面；“联系我”使用 `mailto:` 或复制文本等纯前端交互。→ AC-005, AC-006, AC-009

## 5. 核心逻辑

### 5.1 About 内容读取与兜底策略 → AC-006, AC-007, AC-010

**触发条件**：构建期/请求期（静态输出时等价于构建期）渲染 About 页面。

**处理流程**：
1. 读取 `content/about/about.json`
2. 解析出 About 页面需要的字段（name、tagline、bio、now、skills、contact 等）
3. 对缺失字段做兜底：
   - 头像缺失：渲染“圆形占位（首字母/通用图形）”，不影响标题与正文排版 → AC-007, AC-010
   - 联系方式全部缺失：渲染明确提示文案（例如“暂无公开联系方式”），不渲染无效链接 → AC-006, AC-010
   - Now 为空：该模块可隐藏或显示“近期暂无更新”类轻量占位（二选一，见第 7 节决策）→ AC-010

## 7. 技术决策

### 内容源格式选择：JSON vs Markdown

**背景**：About 的定位是“简短介绍型”，但包含多个固定模块与“缺失字段兜底”需求（头像/联系方式/Now）。→ AC-006, AC-007, AC-010

**选项**：
- A: `content/about/about.json`（推荐）— **优势**：天然结构化，模块字段清晰；缺字段兜底更简单；不引入 MDX 渲染链路。**代价**：纯文本排版灵活性略低（但本期不需要复杂排版）。
- B: `content/about/index.md(x)` — **优势**：写作体验好；自由排版。**代价**：结构化字段（skills/contact/CTA）要靠 frontmatter + 约定，兜底处理更易出歧义；需要引入/完善 MDX 解析链路（当前仓库还未落地）。

**结论**：选择 A（JSON）。核心理由：对本期“固定模块顺序 + 缺失字段兜底”的可控性更强，且不引入新的内容渲染基础设施。→ AC-008, AC-006, AC-007, AC-010

### Now 模块为空时：隐藏 vs 占位

**背景**：Now 属于“可选信息”，缺失时必须保证页面仍可阅读且结构不崩坏。→ AC-010

**选项**：
- A: 隐藏 Now 模块 — **优势**：更干净；减少无意义信息。**代价**：模块顺序会出现“跳过”，但整体仍可读。
- B: 显示轻量占位（如“近期暂无更新”）— **优势**：结构更稳定；访客不会疑惑为什么缺一段。**代价**：可能显得空。

**结论**：选择 A（隐藏 Now 模块），同时保持其他模块顺序不变。理由：站点整体风格“极简文字优先”，空占位更像噪音。→ AC-010, AC-008

## 8. 安全与性能（按需）

**敏感数据处理**：联系信息属于可公开信息；不在页面中渲染任何私密字段（例如手机号/住址）。若未来需要更私密的联系渠道，应以外链/表单服务替代，不直接公开。→ AC-005, AC-006

**性能考量**：页面静态化输出；内容文件体积极小，无需缓存与分页。→ AC-009
