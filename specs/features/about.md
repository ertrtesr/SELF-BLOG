# About（关于我）— 功能需求说明书

## 1. 背景与价值

**为什么做**：访问者打开站点时，需要在 30 秒内理解“作者是谁、在做什么、下一步该看哪里”。About 页提供一个统一、体面、结构清晰的自我介绍入口，并与博客浏览形成自然导流。

**做成什么样**：一个极简文字优先的中文 About 页面，包含关键个人信息与明确的下一步行动（去读博客 / 联系我）。

---

## 2. 范围

### 本次做

- 一个可访问的 About 页面（站点内可导航到达）
- 展示以下模块：
  - 头像 + 姓名/一句话定位（tagline）
  - 简短自我介绍（3-8 句）
  - “我最近在做什么 / Now”小段落
  - 技能/工具栈（精简标签即可）
  - 联系我（提供可直接联系的方式）
  - 引导去博客（Blog）的行动链接/按钮
- About 页面具备基础 SEO 信息（标题/描述），符合“极简文字优先”的整体风格

### 本次不做

- 在线后台/登录后编辑 About — 产品约束（本站不做账号体系）
- 多语言（中英文切换）— 本期仅中文
- 简历式完整履历（教育/经历/项目按时间线罗列）— 本期定位为“简短介绍型”
- 下载 PDF 简历 — 先观察是否真的需要
- 站内联系表单（访客填表提交）— 先用轻量联系渠道即可

## 4. 需求详述

### 4.1 核心流程

1. 访客从站点导航进入 About 页面（或通过链接直接访问）
2. 访客浏览“头像/定位 + 简介 + Now + 技能”来快速建立对作者的整体认知
3. 访客选择下一步行动：去博客继续阅读，或通过“联系我”直接触达作者

### 4.2 交互规则

- About 页面信息层级清晰，优先展示“我是谁/我在做什么/下一步去哪”
- “去博客”的入口在首屏或接近首屏位置可见（不要求固定在同一位置，但需易发现）
- “联系我”提供可直接触达的方式，且不会让访客陷入不确定（例如：点击后无反馈）

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | About 页面必须可通过站点内导航到达（不能是隐藏页） |
| BR-002 | About 页面内容模块按固定顺序呈现：头像/定位 → 简介 → Now → 技能 → 联系我 → 去博客 |
| BR-003 | 当某个可选信息缺失时（例如头像未提供），页面仍应可正常阅读且结构不崩坏 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 站点内可导航到 About
  - **Given** 访客打开站点任意页面
  - **When** 访客使用站点导航寻找“About/关于我”入口并点击
  - **Then** 访客进入 About 页面且页面主要内容可见

- [ ] **AC-002**: About 页面首要信息一目了然
  - **Given** 访客进入 About 页面
  - **When** 访客浏览首屏
  - **Then** 访客能看到“头像 + 姓名/一句话定位（tagline）”与简短自我介绍，并能理解作者基本定位

- [ ] **AC-003**: Now 与技能模块可被清晰扫读
  - **Given** 访客在 About 页面向下浏览
  - **When** 访客看到“Now”和“技能/工具栈”模块
  - **Then** 两个模块有清晰的标题与内容区块，访客可快速扫读其重点信息

- [ ] **AC-004**: About 页可引导去博客
  - **Given** 访客在 About 页面
  - **When** 访客点击“去博客/阅读博客”等引导入口
  - **Then** 访客进入博客列表页（或博客入口页面），且导航/返回行为符合常见网页预期

- [ ] **AC-005**: About 页提供“联系我”并可直接触达
  - **Given** 访客在 About 页面看到“联系我”
  - **When** 访客点击其中一种联系渠道（例如邮箱）
  - **Then** 访客能直接发起联系动作（例如打开邮箱客户端准备发送，或获得明确可复制的联系信息）

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-006**: 联系信息缺失时的兜底呈现
  - **Given** About 页面未提供任何可用的联系渠道
  - **When** 访客浏览到“联系我”区域
  - **Then** 页面以明确文案提示“暂无公开联系方式/稍后补充”等，不出现空白区域或断裂的交互控件

- [ ] **AC-007**: 头像缺失时页面仍可体面展示
  - **Given** About 页面未提供头像
  - **When** 访客打开 About 页面
  - **Then** 页面仍展示姓名/一句话定位与后续内容，整体排版不出现明显错位或“坏掉”的视觉效果

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-008**: About 页面模块顺序与完整性 ← BR-002
  - **Given** 访客在 About 页面从上到下浏览
  - **When** 访客依次阅读页面内容
  - **Then** 页面按“头像/定位 → 简介 → Now → 技能 → 联系我 → 去博客”的顺序呈现，且各模块均有清晰标题或区分方式

- [ ] **AC-009**: About 页面作为站点公开入口可被访问 ← BR-001
  - **Given** 访客未进行任何登录
  - **When** 访客通过站点导航进入 About 或直接访问 About 链接
  - **Then** About 页面可正常访问，不出现“需要登录/无权限”等阻断

- [ ] **AC-010**: 缺失可选信息时页面仍可阅读 ← BR-003
  - **Given** About 页面缺失某一项可选信息（例如头像或 Now 内容为空）
  - **When** 访客浏览 About 页面
  - **Then** 页面信息结构仍完整可读，不出现明显空洞或让人困惑的占位内容
