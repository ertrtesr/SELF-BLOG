# 项目开发路线图 (Development Roadmap)

## 0. 策略概览
*   **开发策略**: 先打好“内容数据模型 + 渲染链路 + 页面骨架”的地基，再完成三大板块（About/Projects/Blog）的可用闭环；随后补齐 SEO 与站内搜索；最后实现“本地可视化编辑 + 一键发布”以完成端到端发布链路。
*   **MVP 目标**: 在 Vercel 可稳定部署的前提下，实现“关于我 / 作品集 / 博客”三大板块可导航与可阅读；具备基础 SEO（meta/OG/sitemap）；具备站内搜索（至少标题+摘要）；并完成一次“本地可视化编辑 + 一键发布”上线流程。

> 进度检测：当前仓库尚未出现 `src/`、`src/modules/*`、`content/` 等实现目录，因此以下任务默认均为未完成 `[ ]`。

## 1. 里程碑一：MVP (核心闭环)
> 目标：实现最基础的业务流程，完成从 0 到 1 的闭环。

- [ ] **Phase 1.1: 基础设施 (Infrastructure)**
    *   **模块**: `Common` / `Content`
    *   **说明**: 初始化 Next.js + TypeScript + Tailwind + 代码规范落盘；建立 `content/` 内容源与构建期解析/校验能力；站点全局布局（极简排版）与导航骨架。
    *   **执行提示词**:
        *   “按 `specs/项目结构.md` 初始化目录，并创建 Next.js App Router 的基础页面骨架（不做业务细节）。”
        *   “实现构建期内容读取与 frontmatter 校验（fail-fast），并产出类型安全的内容模型。”

- [ ] **Phase 1.2: 关于我 (About)**
    *   **模块**: `about`
    *   **说明**: About 页面可用，简历信息结构化展示，导航联动到 Projects/Blog。
    *   **执行提示词**: “开发 About 模块：定义数据结构（教育/经历/技能），实现页面渲染与基础排版。”

- [ ] **Phase 1.3: 作品集 (Projects)**
    *   **模块**: `projects`
    *   **说明**: 项目列表页（卡片）可用；可选：项目详情页（按 MVP 需要决定是否启用）。
    *   **执行提示词**: “开发 Projects 模块：实现项目列表读取、排序、卡片渲染；如启用详情页，补充 slug 路由与详情模板。”

- [ ] **Phase 1.4: 博客 (Blog)**
    *   **模块**: `blog`
    *   **说明**: 文章列表 + 文章详情（MDX）可用；基础排版与代码高亮可按需逐步引入。
    *   **执行提示词**: “开发 Blog 模块：实现文章列表页、`/blog/[slug]` 详情页与 MDX 渲染链路。”

- [ ] **Phase 1.5: 基础 SEO (MVP 必选)**
    *   **模块**: `Common`
    *   **说明**: 全站 meta/OG 生效；生成 sitemap/robots；关键页面具备可分享链接体验。
    *   **执行提示词**: “为 About/Projects/Blog 页面补齐 Metadata，并生成 sitemap 与 robots（符合 Vercel/Next.js 部署）。”

- [ ] **Phase 1.6: 站内搜索 (MVP 必选)**
    *   **模块**: `search`
    *   **说明**: 构建期生成索引（至少标题+摘要），前端可检索文章/作品；无结果与空内容场景不崩溃。
    *   **执行提示词**: “实现构建期索引生成脚本 + 前端搜索页面/组件，并覆盖空索引场景。”

- [ ] **Phase 1.7: 本地可视化编辑 + 一键发布 (MVP 必选)**
    *   **模块**: `publishing`
    *   **说明**: 本地可视化编辑器可编辑 `content/`；一键发布完成“校验→提交→推送→Vercel 自动部署”；生产环境不可访问编辑器入口。
    *   **执行提示词**:
        *   “实现仅开发环境启用的本地编辑器路由 `/(local)/editor`，并将内容写回 `content/`。”
        *   “实现一键发布脚本（格式化/校验/提交/推送），并在 README 中写清发布流程。”

## 2. 里程碑二：功能完善 (Feature Complete)
> 目标：补充辅助功能，提升用户体验。

- [ ] **Phase 2.1: 内容体验增强**
    *   **模块**: `blog` / `projects`
    *   **说明**: 标签/分类（可选）、文章目录（可选）、代码块主题统一、图片/图注规范。
    *   **执行提示词**: “在不引入后端的前提下，为 MDX 内容补充标签/目录/媒体规范，并保持极简风格。”

- [ ] **Phase 2.2: SEO 完善**
    *   **模块**: `Common`
    *   **说明**: 更完善的 OG（按文章/项目动态生成可选）、canonical、结构化数据（可选）。
    *   **执行提示词**: “在 Next.js Metadata 体系下补齐 canonical 与（可选）结构化数据，并确保 sitemap 覆盖动态内容。”

- [ ] **Phase 2.3: 搜索体验增强**
    *   **模块**: `search`
    *   **说明**: 搜索范围扩展（正文可选）、高亮（可选）、快捷键与最近搜索（可选）。
    *   **执行提示词**: “在不引入服务端状态的前提下增强搜索体验，并保持索引体积可控。”

## 3. 里程碑三：运营与优化 (Ops & Optimization)
> 目标：性能优化、可观测性、长期维护能力（不引入在线后台与账号体系）。

- [ ] **Phase 3.1: 性能与体验**
    *   **模块**: `Common`
    *   **说明**: 关键页面性能预算（LCP/TTFB）落地；图片优化；缓存策略（ISR/静态化策略）收敛。
    *   **执行提示词**: “为核心页面定义并实现性能预算，优化渲染与资源加载策略。”

- [ ] **Phase 3.2: 内容规模化与维护**
    *   **模块**: `Content` / `publishing`
    *   **说明**: 内容校验规则体系化（必填字段、链接有效性、slug 冲突）；发布前检查更完善。
    *   **执行提示词**: “建立内容校验规则集与本地发布前检查清单，确保发布可重复、可回滚。”

---
**使用指南**:
1.  **功能开发**: 按照顺序，依次开发每个 Phase。
2.  每完成一个 Phase，请回来勾选进度，并在提交信息中使用对应 scope（见 `specs/开发规范.md`）。
